# 팀 작업 계획

- 팀명: kkirikkiri-product-20260721-1957-003f
- 목표: 러시아워(Rush Hour) 보드게임을 웹 게임으로 만드는 프로젝트의 게임 기획/규칙 설계/구현 방법을 전문가 팀이 토론·분석해서 종합 기획 리포트로 보고
- 생성 시각: 2026-07-21

## 사전 리서치 (완료, 팀원에게 전달됨)
- 러시아워 규칙: 6x6 보드, 자동차 12(1x2)+트럭 4(1x3)+빨간차 1, 직선 슬라이드만 가능, 40~155개 난이도별 퍼즐, PSPACE-complete
- Kenney Car Kit: 40+ 3D 모델(OBJ/FBX/glTF), CC0 라이선스, 승용차/트럭/밴, Godot/Unity/Unreal/웹(Three.js) 호환

## 팀 구성
| 이름 | 역할 | 모델 | 담당 업무 |
|------|------|------|----------|
| leader (메인 세션) | 팀장 | Opus | 계획/배분/검증/통합/최종 리포트 |
| game-systems-designer | 게임 시스템 디자이너 (Analyst) | Opus | 보드 크기 가변성, 난이도 시스템, 레벨 생성/퍼즐 데이터 관리 방식 설계 |
| technical-architect | 기술 아키텍트 (Builder) | Opus | 규칙 구현 로직(이동검증/충돌감지/승리조건), Kenney 에셋 적용 방식, 기술 스택/아키텍처 |
| critic | 검증자 (Critic) | Opus | 두 전문가 결과물 독립 검토 — 리스크, 엣지케이스, 라이선스/구현 난이도 반박 |

## 태스크 목록
- [ ] 태스크 1: 게임 시스템 설계 (보드/난이도/레벨데이터) → game-systems-designer
- [ ] 태스크 2: 기술 구현 설계 (규칙 로직/에셋 적용/스택) → technical-architect
- [ ] 태스크 3: 독립 검증 (두 결과물 반박 + 리스크 도출) → critic
- [ ] 태스크 4: 결과 통합 + 최종 리포트 작성 → leader

## 주요 결정사항
- 4개 논의 주제(기획/규칙구현/에셋적용/기술스택)를 2개 전문가 역할로 분업 (디자이너=기획+난이도, 아키텍트=구현+에셋+스택)
- 별도 인터뷰 확인 없이 추천 옵션(종합 기획 리포트, 제대로 된 깊이)으로 진행 — 사용자가 인터뷰 질문을 스킵하고 "계속해" 요청
- 이 세션 환경은 TeamCreate/Task 기반 Agent Teams API가 없어 Agent 툴(subagent) 기반으로 병렬 실행하도록 적응
